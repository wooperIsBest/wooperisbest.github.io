Tiny BASIC Interpreter

Sample inputs and outputs for
the language interpreter.
This was my third attempt, in a row, at making a programming language (with the first two, I had no idea what I was doing). It uses a simple grammar based mostly on the original Tiny BASIC, with some slightly different functions. Its capabilties include:
  • A basic I/O shell, where each command is run as the user inputs it
  • Program memory, stored when the user types a line number before the command
  • The ability to write to program memory from a file
  • Several basic commands, including variables, INPUT, IF statements, and GOTO numbers
  • A rudimentary error message system
The process of developing this project taught me a lot about how programming languages take what the user inputs and converts it into runnable instructions. While this is an interpreted language, I believe I understand how these concepts can be applied to compilation.

Something I wish I had done differently is choosing a different language to write this in; while I enjoy programming in C, I found myself worrying about memory almost as much as the program's actual functionality (Side note from future me: this is not purely C's fault. I should have been smarter about creating a better memory interface for my needs).

The program takes BASIC-like code as an input and goes through three main steps to interpret their meaning:
  1. Lexical Analysis, otherwise known as tokenizing, converts the text into a series of useable types of data, including numbers, operators, and keywords.
  2. Parsing takes these tokens and organizes them into a structured parse tree, which consists of larger nodes like expressions that then consist of subnodes such as numbers and operators.
  3. Finally, the Interpreter takes these "trees" and runs certain instructions with the given values.
I would love to revisit this project in the future and add several features. First, I would like to give it some more advanced functionality, and a more complete syntax. Additionally, I would like to add better error messages. I would also like to attempt making a compiled language.

Source code for an example program, which lists a Fibonacci-like sequence numbers starting with A, B, A+B, ..., which can be run in the interpreter, is given below.

PRINT "A's ", "value"
INPUT A
PRINT "B's ", "value"
INPUT B
PRINT "Max ", "value"
INPUT C

PRINT A
PRINT B
LET A = A + B
LET B = A + B

IF A < C THEN GOTO 7
